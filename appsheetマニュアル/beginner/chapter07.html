7-1 データ・テーブルのトラブル

スプレッドシートやテーブル設定に関するよくあるトラブルと対処法です。

🔴 行を追加しても AppSheet に反映されない
💡 原因

スプレッドシートで直接行を追加した場合、AppSheet 側でデータが自動同期されないことがあります。また、ヘッダー行の列名が AppSheet 側と一致していない場合にも起きます。

✅ 対処法
1

AppSheet エディタ右上の「Sync」ボタン（🔄アイコン）をクリックします。

2

それでも反映されない場合は、Data > Tables から対象テーブルを開き「Regenerate Structure」を実行します。

3

スプレッドシートのヘッダー行（1行目）が AppSheet のカラム名と一致しているか確認します。

「Regenerate Structure」を実行すると、カラム設定（Type、Label など）がリセットされることがあります。実行前に設定をメモしておきましょう。
🔴 「Add a new row to another table」で行が追加されない
💡 原因

キー列（KEY）に値が設定されていないことが最も多い原因です。AppSheet では、キー列が空のまま行を追加しようとするとエラーになります。

✅ 対処法
1

追加先テーブルのキー列（例：貸出ID）に Initial value として UNIQUEID() を設定します。

2

Action の「Set these columns」からキー列の行を削除します（キー列は Initial value で自動生成されるため、Action 側で設定すると競合します）。

3

Sync して再度 Action を実行し、行が追加されるか確認します。

この問題は本マニュアルの第3章で設定する「貸出ID の Initial value」が漏れている場合に発生します。第3章 セクション3-3 を確認してください。
🔴 Ref（参照）列でドロップダウンが表示されない
💡 原因

カラムの TYPE が Ref に設定されていないか、参照先テーブルの指定が間違っている可能性があります。

✅ 対処法
1

Data > Columns で対象カラムの TYPE を確認し、Ref に変更します。

2

「Source table」が正しいテーブル名を指しているか確認します。

3

参照先テーブルに少なくとも1行のデータが存在することを確認します。

🔴 日付や数値がおかしな形式で表示される
💡 原因

スプレッドシートのセル書式と AppSheet のカラム TYPE が一致していません。たとえばスプレッドシートでは「文字列」なのに AppSheet では「Date」に設定されている場合などです。

✅ 対処法
1

スプレッドシート側でセルの書式を正しく設定します（日付なら「日付」、数値なら「数値」）。

2

AppSheet の Data > Columns でカラムの TYPE が適切か確認します。

3

Google スプレッドシートの地域設定を確認します。スプレッドシートを開き、「ファイル」>「設定」>「全般」>「地域」 が正しいか確認してください。AppSheet 側では Data > Tables > 対象テーブル > Localization > Data locale がスプレッドシートの地域と一致しているか確認します。

AppSheet の Data locale は通常、スプレッドシートの地域設定から自動取得されます。手動で変更した場合や、スプレッドシートの地域設定を後から変えた場合は、AppSheet エディタでテーブルを開き直すと自動的に再取得されます。
新エディタ（New Editor）をお使いの場合：新しいバージョンのエディタでは、テーブル設定画面に「Localization」セクションが表示されない場合があります。Data locale を確認・変更するには、旧エディタ（Legacy Editor）に切り替えてください。新エディタの左下または設定メニューにある「Switch to legacy editor」から切り替えが可能です。旧エディタでは Data > Tables > 対象テーブル > Localization > Data locale で確認できます。

7-2 ビュー・表示のトラブル

アプリのビュー（画面表示）に関するトラブルです。

🔴 ビューにデータが1件も表示されない
💡 原因

ビューの「Row filter condition」や「Slice」の条件が厳しすぎて、該当するデータが0件になっている可能性があります。

✅ 対処法
1

UX > Views で対象ビューを選択し、「Row filter condition」に式が入っている場合は一旦削除してデータが表示されるか確認します。

2

Slice を使用している場合は、Data > Slices でフィルタ条件を見直します。

3

Security Filter が設定されている場合は、Settings > Security で条件を確認します。

🔴 フォーム画面で一部のカラムが表示されない
💡 原因

カラムの「Show?」が OFF になっているか、「Show_If」に条件式が設定されていて条件を満たしていない場合があります。

✅ 対処法
1

Data > Columns で対象カラムの「Show?」にチェックが入っているか確認します。

2

「Show_If」に条件式が入っている場合はその条件を確認します。

3

キー列や計算列（App formula が設定されているカラム）はフォームに表示されない仕様です。意図した動作かどうか確認してください。

🔴 画像が表示されず壊れたアイコンになる
💡 原因

画像カラムの TYPE が Image になっていないか、画像ファイルの URL やパスが正しくありません。

✅ 対処法
1

カラムの TYPE を Image に変更します。

2

画像の URL が有効であることをブラウザで確認します。

3

Google ドライブの画像は「リンクを知っている全員が閲覧可能」に設定します。

7-3 アクション・オートメーションのトラブル

Action やオートメーション（Bot）に関するトラブルです。第5章で設定した貸出登録アクションやメール通知でよく発生します。

🔴 アクションボタンが表示されない
💡 原因

Action の「For a record of this table」が正しいテーブルを指していないか、「Only if this condition is true」の条件式で非表示になっている場合があります。

✅ 対処法
1

Actions 画面で対象の Action を開き、「For a record of this table」がボタンを表示したいテーブル（例：備品マスタ）になっていることを確認します。

2

「Only if this condition is true」に式がある場合はその条件を確認するか、一旦削除してボタンが表示されるかテストします。

3

Action 設定画面の 「Position」 が 「Hide」 になっていないか確認します。「Hide」が選択されていると、どのビューにもボタンが表示されません。

🔴 メール通知が届かない
💡 原因

プロトタイプ状態では、メール通知はアプリ作成者（オーナー）にしか届きません。また、Automation の Event や Process の設定ミスも考えられます。

✅ 対処法
1

まず Manage > Monitor > Audit History で Automation が実行されているか確認します。

2

プロトタイプ状態の場合：To 欄のメールアドレスがアプリオーナーのアドレスと一致しているか確認します。

3

迷惑メールフォルダも確認してください。AppSheet からのメールは noreply@appsheet.com から送信されます。

プロトタイプ状態の制限：プロトタイプ状態（無料）では、オートメーションのメール送信先はアプリのオーナー（作成者）のみに限定されます。他のユーザーにメールを送るにはデプロイ（有料プラン）が必要です。
🔴 メールの件名に列名がそのまま表示される（例：[備品ID]）
💡 原因

Email Step の Subject や Body で & を使った式の結合を記述しています。Email テンプレートでは式ではなくプレースホルダー形式を使う必要があります。

✅ 対処法
項目修正前（誤り）修正後（正しい）
Subject"【備品貸出通知】" & [備品ID] & " が貸し出されました"【備品貸出通知】<<[備品ID]>> が貸し出されました
Email Step の Subject と Body では、& による式の結合は使えません。カラムの値を埋め込むには <<[カラム名]>> の形式で記述してください。詳しくは第5章を参照してください。

7-4 デプロイ・共有のトラブル

デプロイやユーザー共有に関するトラブルです。

🔴 デプロイチェックでエラーが出る
💡 原因

デプロイチェック（Manage > Deploy）ではアプリの構成を検証します。エラーがあるとデプロイできません。

✅ よくあるエラーと対処法
エラーメッセージ対処法
Table '○○' must have a key column対象テーブルにキー列を設定する。Data > Columns でいずれかの列の「KEY?」を ON にします。
Invalid expression in ...指定された箇所の式（Formula）を確認し、構文エラーを修正します。
Column '○○' is required but has no Initial valueREQUIRED が ON のカラムに Initial value または App formula を設定します。
Security: No signed-in user restrictionSettings > Security で「Require user signin」を ON にします。
デプロイチェックの結果が Warning（黄色）であれば、デプロイは可能です。Error（赤）の場合のみデプロイがブロックされます。
🔴 共有したユーザーがアプリを開けない
💡 原因

ユーザーの Google アカウントがスプレッドシートへのアクセス権を持っていない、またはアプリの共有設定にユーザーのメールアドレスが登録されていない場合に発生します。

✅ 対処法
1

AppSheet エディタの Users > Users にユーザーのメールアドレスが登録されているか確認します。

2

Google スプレッドシートの共有設定で、ユーザーに「編集者」または「閲覧者」権限が付与されているか確認します。

3

ユーザーが正しい Google アカウントでログインしていることを確認します。

🔴 「This app is in prototype mode」と表示される
💡 原因

これはエラーではありません。アプリがまだデプロイされておらず、プロトタイプ状態であることを示すメッセージです。

✅ 対処法

プロトタイプ状態でもアプリの基本機能は利用できます（最大10ユーザー）。無料で利用し続ける場合は、このメッセージは表示されたままになりますが、機能に影響はありません。デプロイ（有料プラン）すると、このメッセージは消えます。詳しくは第6章 セクション6-1 を参照してください。

学習・検証目的であればプロトタイプ状態で十分です。慌ててデプロイする必要はありません。

7-5 トラブルシューティングの基本手順

問題が発生したとき、以下の手順で原因を切り分けると効率的に解決できます。

■ 基本の5ステップ

1

Sync する ─ AppSheet エディタ右上の 🔄 ボタンをクリックし、最新データを取得します。多くの問題はこれだけで解消します。

2

エラーメッセージを読む ─ 画面上部やプレビュー画面に表示されるエラーメッセージを確認します。赤い通知バーに具体的な原因が記載されていることが多いです。

3

Audit History を確認する ─ Manage > Monitor > Audit History で、いつ・何が起きたかを時系列で確認できます。Action や Automation の実行結果もここに記録されます。

4

Expression Assistant で式を検証する ─ 式（Formula）のエラーが疑われる場合は、カラム設定画面の式入力欄で Expression Assistant を使い、構文エラーや型の不一致を確認します。

5

スプレッドシートを直接確認する ─ AppSheet を介さずに Google スプレッドシートを直接開き、データの状態（空行、ヘッダーのずれ、書式の不一致など）を確認します。

■ 確認チェックリスト

問題が解決しない場合は、以下のチェックリストを上から順に確認してください。

  • Sync（同期）は実行しましたか？
  • スプレッドシートのヘッダー行（1行目）は正しいですか？
  • カラムの TYPE（型）は適切に設定されていますか？
  • KEY 列に Initial value（UNIQUEID() など）は設定されていますか？
  • REQUIRED が ON のカラムに値は入っていますか？
  • Ref 列の参照先テーブルは正しいですか？
  • Action の「For a record of this table」は正しいテーブルですか？
  • Automation の Event 設定（テーブル名、条件）は正しいですか？
  • 式（Formula）に構文エラーはありませんか？
  • ブラウザのキャッシュをクリアして再読込しましたか？
多くのトラブルは「Sync」と「エラーメッセージの確認」の2ステップで解決します。焦らず、まずはこの2つを試してみましょう。

■ 次のステップ：中級編へ進むために

本マニュアルの備品管理アプリが完成したら、次のテーマに挑戦してみましょう。

テーマ内容
スライス（Slice）テーブルのデータを条件でフィルタし、特定のビューで表示するサブセットを作成します。「貸出中の備品だけ表示」などの絞り込みに使えます。
条件付き書式（Format Rules）ステータスに応じてセルの色を変えるなど、視覚的にわかりやすい画面を作れます。
グラフ表示（Chart View）データを棒グラフや円グラフで可視化します。貸出件数の推移や備品カテゴリの内訳表示に活用できます。
セキュリティフィルタログインユーザーに応じて、表示されるデータを自動的に制御します。部署ごとの閲覧制限などに使います。
承認ワークフロー承認フローの構築や、他の Google サービスとの連携を設定します。中級編で詳しく学びます。
本マニュアルで作成した備品管理アプリは、上記の中級テーマを試すベースとしてそのまま使えます。まずはスライスや条件付き書式から試してみると、AppSheet の表現力が広がることを実感できるはずです。
